This is a sample blog post.

When you want to publish a new article, create another Markdown file in `blog/content` with front matter at the top. GitHub Pages will render that file into a page, and the blog index will pick it up automatically.

## Recommended fields

- `layout: post`
- `title: ...`
- `date: YYYY-MM-DD`
- `tags:` as an optional list
- `excerpt:` for the card on the blog page

You can keep the rest of the file in normal Markdown.
